# Contributing to Audotics

Thank you for considering contributing to Audotics! This guide outlines the process for contributing to the project and helps you get started.

## Code of Conduct

Audotics is committed to fostering an open and welcoming environment. By participating in this project, you agree to abide by our Code of Conduct.

## How Can I Contribute?

There are many ways to contribute to Audotics:

### Reporting Bugs

If you find a bug in the codebase, please report it by opening an issue in our issue tracker. Before submitting a bug report, please check if the issue has already been reported. When reporting a bug, include as many details as possible:

- A clear and descriptive title
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Screenshots or error messages
- Environment details (OS, browser, etc.)

### Suggesting Enhancements

We welcome suggestions for enhancing Audotics. To suggest an enhancement:

1. Open a new issue
2. Use a clear and descriptive title
3. Provide a detailed description of the suggested enhancement
4. Explain why this enhancement would be useful to most Audotics users

### Pull Requests

We actively welcome pull requests:

1. Fork the repository
2. Create a new branch for your feature/fix
3. Make your changes
4. Run tests to ensure your changes don't break existing functionality
5. Submit a pull request

### First-Time Contributors

If you're a first-time contributor, look for issues labeled "good first issue" or "help wanted". These issues are specifically earmarked for first-time contributors.

## Development Workflow

### Setting Up the Development Environment

Follow the [setup instructions](../setup/README.md) to get your development environment ready.

### Coding Standards

We maintain a high standard of code quality. Please follow these guidelines:

- Follow the existing code style
- Write clear, readable, and maintainable code
- Add comments for complex logic
- Include appropriate tests for your changes
- Update documentation as needed

### Commit Guidelines

- Use clear and meaningful commit messages
- Reference issue numbers in commit messages
- Keep commits focused and atomic
- Squash multiple commits if they relate to a single change

### Testing

All contributions must pass existing tests and include new tests for added functionality:

- Run frontend tests: `cd frontend && npm test`
- Run backend tests: `cd backend && npm test`
- Run end-to-end tests: `npm run test:e2e`

### Documentation

- Update documentation for any changed functionality
- Document new features thoroughly
- Ensure README files and code comments are up to date

## Review Process

All submissions require review:

1. A maintainer will review your pull request
2. Feedback may be provided, requiring changes
3. Once approved, your contribution will be merged

## Recognition

All contributors are recognized in our CONTRIBUTORS.md file. We appreciate all forms of contribution!

## Need Help?

If you need help with anything related to contributing, please reach out by:

- Opening an issue labeled "question"
- Contacting one of the maintainers directly

Thank you for contributing to Audotics!
